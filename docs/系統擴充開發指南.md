# 🤖 RAG 系統擴充開發指南 (For AI Assistant)

> **[給 AI 助手的系統指令]**
> 你現在正在協助擴充一個基於 LangGraph 與 LangChain 建構的異步 (Async) RAG 客服系統。
> 在生成任何程式碼之前，請務必詳細閱讀以下架構規範。**絕對禁止**為了新增單一功能而修改核心圖表邏輯 (`graph/builder.py` 或 `main.py`)。本系統嚴格遵循「設定檔驅動 (Config-Driven)」與「工廠模式 (Factory Pattern)」。

## 1. 架構核心思想

本系統高度解耦，主要分為以下模組資料夾，每個資料夾都是一個獨立的工廠：

* `llms/`: 負責提供語言模型實例。
* `retrievers/`: 負責提供檢索器實例。
* `embeddings/`: 負責提供文字轉向量模型實例。
* `memory/`: 負責提供對話記憶存儲 (Checkpointer) 實例。

## 2. 模組開發四大鐵則 (Golden Rules)

1. **從不硬編碼 (No Hardcoding)**：所有的 API Key、模型名稱、連線網址、超時設定等，都必須從傳入的 `config: dict` 中讀取，或使用 `os.getenv` 讀取環境變數。
2. **單一職責 (Single Responsibility)**：新增一個供應商（例如 OpenAI, Anthropic, Qdrant），就必須建立一個全新的 `.py` 檔案。
3. **註冊表機制 (Registry Pattern)**：寫完新的 `.py` 檔案後，**只能**修改該模組資料夾下的 `__init__.py`，將新功能匯入並加入 `REGISTRY` 字典或判斷式中。
4. **相容性 (Compatibility)**：回傳的物件必須相容於 LangChain/LangGraph 的標準介面（例如 `BaseChatModel`, `Embeddings`, `BaseCheckpointSaver`）。

## 3. 各模組實作規範與範例

當使用者要求「新增 XXX 支援」時，請依照以下對應的介面進行實作：

### 3.1 新增語言模型 (LLMs)

* **目標資料夾**: `llms/`
* **實作規範**:
1. 建立 `[provider]_model.py`。
2. 建立一個命名為 `build_[provider]_llm(config: dict)` 的函數。
3. 必須回傳 LangChain 的 `BaseChatModel` (如 `ChatOpenAI`, `ChatAnthropic`)。


* **註冊方式**: 在 `llms/__init__.py` 中匯入該函數，並加入 `REGISTRY` 字典。

### 3.2 新增檢索器/資料庫 (Retrievers)

* **目標資料夾**: `retrievers/`
* **實作規範**:
1. 建立 `[provider]_store.py`。
2. **必須繼承** `from .base import BaseRetriever`。
3. 必須實作 `def setup(self):` 進行初始化與讀取 `self.config`。
4. **必須實作非同步方法** `async def aretrieve(self, question: str) -> str:`，並回傳格式化好的字串 (Context)。若查無資料，請回傳字串預設提示。


* **註冊方式**: 在 `retrievers/__init__.py` 中匯入該 Class，並加入 `REGISTRY` 字典。

### 3.3 新增向量模型 (Embeddings)

* **目標資料夾**: `embeddings/`
* **實作規範**:
1. 建立 `[provider]_embed.py`。
2. 建立一個命名為 `build_[provider]_embedding(config: dict)` 的函數。
3. 必須回傳 LangChain 的 `Embeddings` 介面物件。


* **註冊方式**: 在 `embeddings/__init__.py` 中匯入該函數，並加入 `REGISTRY` 字典。

### 3.4 新增對話記憶體 (Memory)

* **目標資料夾**: `memory/`
* **實作規範**:
1. 若需引入外部資料庫連線（如 Postgres, Redis），建議寫在獨立檔案後匯入。
2. 必須回傳相容於 LangGraph 的 Checkpointer (如 `MemorySaver`, `AsyncSqliteSaver`, `AsyncPostgresSaver`)。


* **註冊方式**: 直接在 `memory/__init__.py` 的 `get_checkpointer(config: dict)` 判斷式中新增 `elif memory_type == "xxx":` 分支。

---

## 4. 給 AI 的輸出要求 (Output Format)

當你（AI）接收到新增模組的任務時，你的回答必須包含：

1. **新增的 Python 檔案內容** (例如 `llms/openai_model.py`)。
2. **`__init__.py` 的修改指示** (明確指出要加入註冊表的程式碼)。
3. **`config.toml` 的修改範例** (展示使用者應該如何在設定檔中啟用這個新模組，包含必要的參數設定)。
4. *(可選)* 提醒使用者需要 `pip install` 的新套件名稱。
