# 20260318 — 配置驅動的 LINE Rich UI（FlexMessage 影片卡片）

## 目標

讓 YouTube 教學影片知識庫（`db_youtube`）的檢索結果能以 LINE FlexMessage 影片卡片呈現，取代純文字回覆。透過 `config.toml` 的 `ui_type` 欄位驅動 UI 類型，實現配置驅動的多模態回覆。

## 核心挑戰

LangGraph `ToolNode` + `StructuredTool` 的 `aretrieve()` 只能回傳 `str`，metadata 無法直接傳遞給下游節點。

**解法**：retriever 在回傳字串尾部以 `===UI_METADATA===` 分隔符嵌入 JSON metadata，agent 的 `execute_tools()` 攔截並剝離後寫入 `state["ui_hints"]`，LLM 看到的仍是乾淨文字。

## 技術設計

### 資料流

```
PGVectorRetriever.aretrieve()
  → 文字 + ===UI_METADATA=== + JSON metadata
  → ToolMessage（含嵌入 metadata）

execute_tools()（agent 子圖內）
  → 攔截 ToolMessage，用 UI_TYPE_MAP 比對
  → 剝離 metadata → state["ui_hints"]
  → LLM 只看乾淨文字

post_process()
  → build_line_messages(answer, ui_hints)
  → 產生 [TextMessage, FlexMessage] → state["response_ui"]

send_response()
  → 直接傳入 message_objects → LINE Reply / Push API
```

### UI 工廠邏輯（`build_line_messages()`）

1. 從 `ui_hints` 篩選 `VIDEO_CARD` 類型
2. 優先取 `url` 欄位（完整 YouTube URL），fallback 到 `source`（裸 video ID）
3. 依 `video_id` 去重（同影片不同片段只出一張卡片）
4. 自動從 video_id 組合縮圖 `https://img.youtube.com/vi/{id}/maxresdefault.jpg`
5. 1 部影片 → `FlexBubble`；2+ 部 → `FlexCarousel`（上限 10 張）
6. 回傳順序：`[TextMessage(answer), FlexMessage]`
7. 無有效影片 → 降級 `[TextMessage(answer)]`

### Reply → Push 降級策略

- `TextMessage`：加前綴 `【系統通知】讓您久等了，以下是您的回覆：`
- `FlexMessage`：原樣保留（不加前綴）

## 不修改的檔案

| 檔案 | 原因 |
|------|------|
| `tools/base_retriever.py` | `aretrieve()` 介面不變（回傳 str），metadata 嵌入由子類別處理 |
| `app.py` | 無需修改，`debounce` 模組內部處理 `response_ui` 傳遞 |
| 其他 agent prompt | 不受影響，`ui_hints` 攔截對 LLM 透明 |
| 非 YouTube 資料庫 | `ui_type` 預設 `TEXT`，行為完全不變 |
