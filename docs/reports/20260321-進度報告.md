# 20260321 — Bug 修復：pre_process 遺失 HumanMessage、摘要品質優化、rewrite_query 暫停

## 目標

修復 `contents are required` 錯誤，同時記錄近期 `rewrite_query` 暫停、摘要格式改進、以及 `pre_process` 摘要上下文強化等變更。

## 1. Bug 修復：pre_process 遺失 HumanMessage

### 問題

執行 `python main.py` 或 LINE Bot 收到訊息時，Gemini API 回傳錯誤：

```
[LangGraph 執行錯誤] contents are required.
```

### 根因

`graph/nodes.py` 的 `pre_process` 函式原本有一行 `messages.append(HumanMessage(content=state["question"]))` 將使用者問題加入 `messages`。該行被移除後，預期由 `rewrite_query` 節點負責注入，但 `rewrite_query` 在 `builder.py` 中被暫時停用。

結果：`messages` 只包含 SystemMessage（前情提要），沒有 HumanMessage，Gemini API 因缺少 user content 而報錯。

### 修復

在 `graph/nodes.py` 的 `pre_process` 函式中，於 `return` 之前加回：

```python
messages.append(HumanMessage(content=state["question"]))
```

## 2. rewrite_query 節點暫停使用

### 變更

在 `graph/builder.py` 中將 `rewrite_query` 節點及其連線註解掉，流程改為 `manage_memory → router` 直連：

```python
# workflow.add_node("rewrite_query", rewrite_query)   # 暫時停用
# workflow.add_edge("manage_memory", "rewrite_query")  # 暫時停用
# workflow.add_edge("rewrite_query", "router")         # 暫時停用
workflow.add_edge("manage_memory", "router")
```

### 目前流程

```
START → pre_process → manage_memory → router → {agents | out_of_domain | human} → merge_answers → update_profile → post_process → END
```

## 3. pre_process 摘要上下文強化

### 變更

`pre_process` 注入的摘要 SystemMessage 增加注意事項，引導 LLM 只參考與當前問題相關的摘要內容，避免歷史話題混入回答：

```python
messages.append(SystemMessage(content=(
    f"[前情提要]\n{summary}\n\n"
    "【注意】以上為歷史對話摘要，可能包含多個不同話題。"
    "請只參考與使用者「當前問題」直接相關的部分，"
    "忽略不相關的歷史話題，避免將不同主題的資訊混入回答。"
)))
```

---

## 4. 摘要格式改進

### 變更

`agents/prompts/summarize_messages.md` 新增第 7 條規則：

> 不同話題之間用空行分隔，每個話題前加上簡短標題（如「鎖舌故障」「APP 設定」），方便後續判斷相關性

搭配第 3 點的摘要上下文強化，讓 LLM 能更精準地從結構化摘要中篩選相關資訊。
