# 20260319 — 架構解耦、配置驅動多模態 UI、rewrite_query 節點引入

## 目標

同步記錄近期三大架構改進：app.py 瘦身解耦、配置驅動多模態 UI 完善、rewrite_query 獨立節點引入。

## 1. 架構解耦：app.py 瘦身

將 `app.py` 的防抖邏輯與 LINE 通訊邏輯拆離，使 webhook 入口僅負責生命週期管理與路由。

| 模組 | 職責 |
|------|------|
| `core/debounce.py` | 訊息防抖層：緩衝合併、LangGraph 流程控制、`response_ui` 傳遞 |
| `core/line_bot.py` | LINE Messaging API 封裝：`send_response()` 支援 Message 物件 / `show_loading` |
| `app.py` | FastAPI webhook 入口：生命週期管理（startup/shutdown）、路由瘦身 |

**效益**：
- 各模組職責單一，易於獨立測試
- `debounce.py` 可被 CLI（`main.py`）複用，不依賴 LINE SDK
- `line_bot.py` 可替換為其他通訊平台（如 Telegram）而不影響核心邏輯

## 2. 配置驅動多模態 UI

透過 `config.toml` 的 `ui_type` 欄位驅動 UI 類型，實現零程式碼的多模態回覆切換。

### 核心元件

| 元件 | 說明 |
|------|------|
| `SupportedUI` enum | `core/constants.py` — 定義 `TEXT`、`VIDEO_CARD` 等 UI 類型 |
| `UI_TYPE_MAP` | `tools/__init__.py` — 工具名稱 → UI 類型映射表 |
| `line_ui_factory.py` | `tools/line_ui_factory.py` — `build_line_messages()` 產生 FlexMessage |

### 資料流

```
Retriever → ===UI_METADATA=== JSON → execute_tools() 攔截 → ui_hints
→ post_process → build_line_messages() → [TextMessage, FlexMessage]
→ send_response() → LINE API
```

### Fallback 降級

- metadata 缺少必要欄位 → 自動降級為純文字（TEXT）
- 單部影片 → FlexBubble；多部影片 → FlexCarousel（上限 10 張）
- Reply Token 過期 → Push API（FlexMessage 原樣保留）

## 3. rewrite_query 節點引入

在 `manage_memory` 與 `router` 之間新增 `rewrite_query` 獨立節點，解決多輪對話中代名詞與意圖遺失問題。

### 管線流程變更

```
舊：manage_memory → router
新：manage_memory → rewrite_query → router
```

### 職責

1. **代名詞解析**：將「它」、「那個」替換為具體品牌/型號（結合 user_profile）
2. **脈絡補齊**：結合前情提要（summary），將過於簡短的問題補齊為完整檢索句
3. **原意保持**：只改寫問題，不回答問題；問題已完整時原樣回傳

### 異動

| 檔案 | 動作 | 說明 |
|------|------|------|
| `agents/prompts/rewrite_query.md` | 新增 | 問題改寫專家 prompt |
| `graph/nodes.py` | 修改 | 新增 `rewrite_query()` 節點函數 |
| `graph/builder.py` | 修改 | 新增節點、調整邊：`manage_memory → rewrite_query → router` |
| `config.toml` | 修改 | `[prompts]` 新增 `rewriter` 路徑 |

### 效益

- Router 收到的是精確檢索句，意圖分類準確率提升
- Agent 不再需要自行推敲代名詞，Prompt 複雜度降低
- 檢索命中率提升（關鍵字更明確）
