# 20260325 — 對話上下文重構、Agent Prompt 優化、DOWNLOAD_CARD 檔案名稱

## 目標

強化多輪對話理解能力（Router 濃縮歷史、Agent 接收上下文）、統一 Agent Prompt 風格與轉接流程、新增說明書下載卡片檔案名稱顯示，並全面同步架構圖至現行 7 Agent 配置。

---

## 1. Router 對話上下文重構

### 問題

Router 只看到當前單句問題，缺乏多輪脈絡，容易在代名詞（「那個」「剛才說的」）或追問場景下誤判意圖。

### 解法

- **Router Prompt 新增「濃縮問題」指令**：Router 在意圖分類前，先將多輪對話壓縮為一句自包含的問題（`condensed_question`），取代直接傳遞歷史 pairs 給 Agent
- **`_extract_recent_pairs`**：從 messages 中提取最近 N 輪 Human/AI 對話，注入 Router 的 system prompt 作為脈絡參考
- **Agent 不再接收歷史 pairs**：改為只接收 Router 濃縮後的單句問題，簡化 Agent 子圖的輸入

### 影響範圍

- `graph/nodes.py` — `router` 節點重構，新增 `_extract_recent_pairs`
- `agents/prompts/router.md` — 新增濃縮問題指令與範例
- `graph/builder.py` — 移除 agent 子圖的歷史 pairs 傳遞

## 2. 轉接真人流程重構

### 問題

- 轉接表單中出現 Agent 道歉語與表單內容重複
- 轉接後所有後續訊息都被誤判為 `transfer_human` 意圖（歷史中殘留的轉接 HumanMessage 干擾 Router）

### 解法

- **`merge_answers`**：轉接時保留 Agent 道歉語作為開頭，再接上轉接表單，消除重複語句
- **`_extract_recent_pairs` 過濾**：自動過濾尾端未配對的 HumanMessage（轉接後的孤立訊息），防止 Router 被殘留的轉接脈絡誤導
- **`transfer_human_form.md`**：精簡表單格式，移除與 Agent 道歉語重複的開場白

### 影響範圍

- `graph/nodes.py` — `merge_answers` 轉接邏輯重寫
- `agents/__init__.py` — Agent 子圖清理 tool 中間訊息
- `agents/prompts/transfer_human_form.md` — 表單精簡
- `app.py` — 轉接狀態管理

## 3. Agent Prompt 統一優化

### 變更

四個專業 Agent（`hardware_technician`、`sales_representative`、`store_assistant`、`app_specialist`）的 Prompt 統一加入：

- **Fallback 策略一致化**：檢索不到時先提供通用建議，再引導使用者補充資訊，最後才提議轉接真人
- **人情味提升**：回覆語氣更自然、加入同理心表達
- **`update_profile` Prompt 中文化**：萃取規則改為中文，新增電話號碼防呆規則（避免將非電話數字誤判為手機號碼）

### 影響範圍

- `agents/prompts/` — 4 個 Agent prompt + `update_profile.md`

## 4. DOWNLOAD_CARD 顯示檔案名稱

### 問題

說明書下載卡片的 title 永遠顯示 fallback 值「說明書檔案」，因為 `kb_gdrive` metadata 沒有 `title` 欄位。

### 解法

- **`line_ui_factory.py`**：
  - `_build_download_bubble_dict` 新增 `filename` 參數，卡片 body 中新增一行灰色小字 `📎 {filename}`
  - `build_line_messages` 中 title fallback 到 `model` 欄位組合（如「DP850 說明書」），filename 組合為 `"{model} 說明書.pdf"`
- **新增 `manual_librarian` Agent**：說明書管理員，使用 `db_manuals` 工具（`ui_type=DOWNLOAD_CARD`）

### 卡片結構

```
┌──────────────────────┐
│ 📄 官方說明書          │  ← header
│ DP850 說明書          │  ← title (from model)
│ 📎 DP850 說明書.pdf   │  ← filename (新增)
│ 點擊下方按鈕即可開啟   │  ← description
│  ┌──────────────┐    │
│  │ 📥 立即下載   │    │  ← button
│  └──────────────┘    │
└──────────────────────┘
```
