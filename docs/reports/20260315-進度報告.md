# 20260315 — User Facts SQL 化（SCD Type 2）

## 目標

本次為 **PostgreSQL 全面整合的最終階段**。系統從 SQLite 到 PostgreSQL 的遷移歷經三個階段，本次完成最後一塊拼圖——User Facts SQL 化：

將使用者「硬事實」（phone, address, device_model, device_brand）從 `.md` 自由格式遷移至 PostgreSQL `user_facts` 表，實作 SCD Type 2 版本控制，解決 LLM 自由格式管理結構化資料的不穩定性與缺乏歷史追蹤的問題。

## 遷移歷程

| 階段 | 內容 | 對應 Commit |
|------|------|------------|
| 第一階段：Memory | 對話記憶（Checkpointer）從 SQLite 遷移至 PostgreSQL，使用 `AsyncPostgresSaver` | `9187e3b` |
| 第二階段：Audit Log | 審計日誌從 SQLite 遷移至 PostgreSQL，實作 `PostgresAuditStorage` | `9187e3b` |
| 第三階段：User Facts | 使用者硬事實新增 PostgreSQL `user_facts` 表，SCD Type 2 版本控制 | `057bba0` |

至此，系統所有持久化資料（對話記憶、審計日誌、使用者事實）均以 PostgreSQL 為預設儲存，SQLite 僅作為回退方案保留。

## 技術設計

### SCD Type 2 寫入流程

```sql
-- 1. 過期舊值（僅在值改變時）
UPDATE user_facts SET is_current = FALSE, end_date = NOW()
WHERE user_id = %s AND attr_key = %s AND is_current = TRUE AND attr_val != %s;

-- 2. 插入新值（冪等，同值不重複插入）
INSERT INTO user_facts (user_id, attr_key, attr_val, is_current, start_date)
SELECT %s, %s, %s, TRUE, NOW()
WHERE NOT EXISTS (
    SELECT 1 FROM user_facts
    WHERE user_id = %s AND attr_key = %s AND attr_val = %s AND is_current = TRUE
);
```

### 降級策略

所有 DB 操作都有 try/except 包裝：
- `init_facts_db` 失敗 → `_facts_conn = None`，print 警告
- `load_facts` 遇到 `_facts_conn is None` 或查詢失敗 → 回傳 `{}`
- `update_fact` 失敗 → print 錯誤，不中斷流程
- `update_profile` JSON 解析失敗 → fallback 整段當 soft profile 存

### 資料分層

| 類別 | 儲存位置 | 內容 | 管理方式 |
|------|---------|------|---------|
| 硬事實 | PostgreSQL `user_facts` | phone, address, device_model, device_brand | SCD Type 2，SQL 直接管理 |
| 軟資訊 | `data/profiles/*.md` | 偏好、過去問題、個性、居住環境等 | LLM 萃取，Markdown 格式 |
