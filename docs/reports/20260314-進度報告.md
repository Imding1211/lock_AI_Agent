# 20260314 — 敏感交易議題強制轉接真人護欄

## 目標

針對涉及報價、金流等敏感商務詞彙的使用者訊息，在 LLM 意圖分類之前即攔截，強制轉接真人客服，避免 AI 產生不當的價格或金流回覆。

## 技術設計

### 攔截機制

```python
# graph/nodes.py — router() 函式內，LLM 呼叫之前
sensitive_keywords = SYSTEM_CONFIG.get("sensitive_keywords", [])
if sensitive_keywords:
    for kw in sensitive_keywords:
        if kw in question:
            answer = await _transfer_tool.generate_form(user_id, extra_text=question)
            return {
                "answer": answer,
                "next_agents": [],
                "history": ["guardrail_triggered", "topic_resolved"],
            }
```

### 設計考量

- **前置於 LLM**：關鍵字掃描在 LLM 意圖分類之前執行，敏感詞彙不進入 LLM 流程
- **複用既有機制**：使用與 `transfer_human` 意圖相同的 `TransferHumanTool.generate_form()` 產生轉接表單
- **可配置**：關鍵字清單完全由 `config.toml` 驅動，無需修改程式碼
- **無新增節點**：`next_agents=[]` 直接走 `Send("merge_answers")` 既有流程

| 場景 | 路徑 | 結果 |
|------|------|------|
| `demo_guardrail`：「這款電子鎖多少錢？可以報價嗎？」 | `guardrail_triggered` → `topic_resolved` | 跳過 LLM，直接產生轉接表單 ✅ |
| 既有場景（產品/故障/訂單/領域外/轉接真人） | 不含敏感詞，正常走 LLM 分類 | 全部通過 ✅ |
