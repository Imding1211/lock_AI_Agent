# 20260313 — Per-DB Embedding 配置與 pgvector 整合

## 目標

將全域 `[embedding]` 區塊移除，Embedding 設定回歸為各 `[[databases]]` 獨立配置（`embedding_provider`、`embedding_model` 等欄位），並將向量儲存從 ChromaDB 遷移至 pgvector（Docker）。

## 異動摘要

### 程式碼

| 檔案 | 動作 | 說明 |
|------|------|------|
| `config.toml` | 修改 | 移除 `[embedding]` 區塊，各 `[[databases]]` 保留 `embedding_provider`/`embedding_model` 等欄位；向量資料庫 `type` 改為 `pgvector`，區塊編號由 14 → 13 |
| `core/config.py` | 修改 | 移除 `embedding` 區塊讀取，不再匯出 `EMBEDDING_CONFIG` |
| `embeddings/__init__.py` | 修改 | `get_embedding()` 要求 config 必須帶 `embedding_provider`，移除全域 fallback |
| `tools/pgvector_store.py` | 新增 | PGVectorRetriever，支援 pgvector 向量資料庫檢索 |
| `tools/__init__.py` | 修改 | REGISTRY 新增 `pgvector` 映射 |

### 文件

| 檔案 | 說明 |
|------|------|
| `docs/assets/architecture.mmd` | 移除 `EMBEDDING_ENGINE` 節點及其連線 |
| `docs/manuals/系統架構與流程圖.md` | Support_Layer 移除 Embedding Factory 節點 |
| `docs/manuals/系統擴充開發指南.md` | 架構總覽移除 `[embedding]` 路徑，§5 改寫為 Per-DB Embedding 設定 |
| `docs/manuals/系統設定檔 (config.toml) 說明文件.md` | 移除 §6 `[embedding]` 說明，§7.A 改為 pgvector 範例，重新編號 |

## 驗證項目

- [x] `config.toml` 無 `[embedding]` 區塊，各 pgvector DB 獨立配置 `embedding_*` 欄位
- [x] `core/config.py` 不載入 `embedding` 區塊，無 `EMBEDDING_CONFIG`
- [x] `get_embedding()` 無 fallback，要求 `embedding_provider` 必填
- [x] 架構圖中無 `EMBEDDING_ENGINE` 節點
- [x] `tools/__init__.py` REGISTRY 含 `pgvector` 映射
